# Documentation examples

This folder holds **filled-in sample documents** for reference. They are **not** templates.

- **Templates** (how to structure new docs, with placeholders and instructions) live under [`docs/templates/`](../templates/).
- **Samples** here illustrate one possible complete outcome for a specific product shape. Do not copy them wholesale into new initiatives; start from the matching template and replace placeholders with your own content.

| Sample | Notes |
| --- | --- |
| [`multi-list/product-requirements-sample.md`](multi-list/product-requirements-sample.md) | Example baseline PRD-style requirements for a multi-list web app. |
| [`multi-list/technical-design-sample.md`](multi-list/technical-design-sample.md) | Example technical design tied to that sample PRD. |
| [`multi-list/design-wireframe-sample.md`](multi-list/design-wireframe-sample.md) | Example wireframe and UX annotation pack for the same initiative. |

For **paste-ready long skeletons** (full section lists you can duplicate in a new file), prefer the skill references:

- Product: `.cursor/skills/product-manager/reference.md`
- Technical design: `.cursor/skills/technical-designer/reference.md`

Wireframing process and output types are described in `.cursor/skills/designer-wireframe/SKILL.md`.
