# Security Policy

## Supported versions

This template is maintained on a best-effort basis. Report vulnerabilities against the latest `main` branch.

## Reporting a vulnerability

- Do not open public GitHub issues for security problems.
- Email the maintainers with:
  - A clear description of the vulnerability
  - Reproduction steps or proof of concept
  - Potential impact
  - Suggested mitigation (if known)

If this repository is copied to start a new project, update this file immediately with the correct security contact for that project.

## Response expectations

- Initial acknowledgment target: within 3 business days
- Triage and severity assessment target: within 7 business days
- Fix or mitigation timeline depends on severity and exploitability
